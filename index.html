The Power Amplifier features a two-channel design with a built-in DSP section based on STM32H7 MCU(ARM Cortex-M7) boasting a sophisticated delay, active crossover, EQ filters, dynamics processing, lockable security settings, output safe limiting, user-selectable input sensitivity and much more, which can be carefully controlled in the server-side dynamic web page embedded in the MCU.

  • Stereo/parallel/bridge-mono mode.
  • Independent protection against shorts, no-load, overload, DC voltage and thermal overload on each channel.
  • Electronically balanced RCA & XLR & 1/4'' TRS inputs/outputs; speakon outputs.
  • Two-channel, 2 x 500-Watt @ 4Ω Power Amplifier.
Floating point library based on VHDL
VHDL Design On FPGA

This library, with a different approach than that of the similar libraries, implements the idea of seed breeding to set up and work with the display.

  • It is only designed for 2X16 LCM, but the library structure is very flexible so the user can easily change it for various types of LCM.
  • With this library, the user can display animation on his/her screen by adding only a few lines of code to the main body of the VHDL program.
  • Other abilities such as setting the position of the cursor ('lcm_gotoxy' function), sending a string to display ('lcm_string'), creating nested control block like a loop in other languages ('subseed_breeding'), creating a proper delay in the display workflow without compromising the performance ('lcm_delay'), etc.
  • Link to GitHub.
ALL-IN-ONE Driver
VHDL Design On FPGA
75%

With this object-oriented pseudo driver written in VHDL language, one will be able to set up a variety of sensors in several control modes based on settling time and reliability. All calculations are performed with the help of a floating-point unit with high accuracy.

  • Each object contains many devices (whether sensors, actuators or deactuators) that function and behave identically.
  • There is no software limit, for the number and type of the objects in this driver.
  • In this driver, one will be capable to set up a variety of actuators/de-actuators in several control modes, based on settling time and practical efficiency.
  • Ability to define various operating modes for each specific object, based on hardware features (e.g. presence or absence of actuators).
  • Ability to define the measurement gap for each object according to the sufficient accuracy and hardware conditions.
  • Optional liquid crystal display support to track the working status of the device.
  • Ability to specify three operating modes ('TURBO', 'NORMAL' and 'SLEEP') for the whole system.
  • All of the basic settings can be done via the UART protocol during startup, by activating the bootloader feature.
  • Ability to define different control methods for each object, based on the sensor operating range or point.
  • Possibility to define a specific exception in the operating range for each object.
  • Ability to properly communicate with the host to adjust and carefully trace all the features based on the standard UART protocol. The user can make the local connection more secure by setting a unique password.
  • Link to GitHub. (Under development!)
Graphic LCD driver for AVR platform
AVR Microcontrollers

An object-oriented pseudo driver (the driver is object-oriented pseudocode written in C language, each object has its own position and size) was developed that supports various fonts such as Calisto-MT, Bradley, Arial, etc.

  • This driver uses optimal memory management to reduce memory consumption by controlling the number of attempts to access memory addresses.
  • It supports object magnification, removing and sending objects without interfering constructively with other visible objects and makes animation possible by sending and removing objects to/from the screen at periodic intervals.

SUDIO Project
DSP Python

Sudio is a digital audio processing library featuring both real-time and non-real-time mix/edit platforms.

The sudio:

  • is written in python and uses scientific computational packages such as NumPy and scipy to manipulate data.
  • can manage various audio streams from audio files or operating systems without any size limitation.
  • is written flexible can used for high level audio processing algorithm and other system factors, which might cause the latency effects.
  • It also can estimate the synchronization and delay of multiple channels.
  • is a comprehensive library for mixing, editing, and recording audio content.
  • used additional cached files to reduce dynamic memory usage and improve performance.
